# Does your startup need more features?

**Author:** Oliver Stafurik
**Date:** October 18, 2024
**Source:** https://www.indiehackers.com/post/9uV9pzyaDIndEdp0Blnb

---

## Main Thesis

The article argues that adding features doesn't always improve products -- it can actually slow down development and adaptability. The author uses the "backpack metaphor" to illustrate how each new feature adds cumulative weight that impairs a startup's ability to pivot and iterate.

## Core Arguments

### Feature Complexity Trade-offs

Features vary in their "weight." Some deliver significant customer value without much codebase complexity, while others impose lasting maintenance burdens. The author emphasizes: "Adding features adds weight to your backpack."

### Real-World Examples

- **Private profiles** at WIP (the author's product): Required changes across all code that displays todos, creating permanent complexity considerations
- **Dark mode**: Seems simple initially but requires all future design changes to include a dark variant, slowing every design decision indefinitely

**Key Principle:** Before adding features, founders should ask: "How much weight will this add to my backpack?"

## Secondary Topics

### Richard Branson on Delegation

The article highlights Branson's philosophy that founders cannot do everything themselves. Delegation allows focus on strategic priorities rather than tasks outside one's expertise.

### Venture Capital Shift

The article notes that the era of abundant, low-cost venture funding has ended. Companies now must demonstrate real impact rather than relying on inflated valuations.

## Actionable Lessons

1. Prioritize staying nimble over feature expansion, especially pre-product-market fit
2. Evaluate features based on complexity burden, not just user appeal
3. Delegate non-core responsibilities to maintain strategic focus
4. Solve core problems deeply rather than addressing multiple problems superficially
