# Are AI assistants making us worse programmers?

**Author:** Rafael Quintanilha
**Date:** November 22, 2024
**Source:** https://www.indiehackers.com/post/CGOOV7OS7Z3Y1YtfPYmg

---

**Core Question:** Whether AI programming assistants degrade developer capabilities.

**Main Thesis:** AI assistants aren't inherently detrimental; their impact depends on how developers employ them strategically.

## Key Lessons

1. **Distinguish Core Skills from Tool Knowledge**
   - Core competencies (algorithms, system design, problem decomposition) remain valuable
   - Tool-specific syntax and framework APIs are transient and less critical
   - Example: Understanding Server-Side Rendering outlasts knowing specific Gatsby configurations

2. **AI Excels at "Bureaucratic" Work**
   - Implementation details that don't require strategic thinking
   - Type conversions, boilerplate code, UI animations
   - Frees cognitive resources for architecture and design decisions

3. **Danger: Over-Reliance Trap**
   - Blindly delegating problems without understanding them wastes time
   - Depletes willpower and requires managing code you didn't author
   - Requires intentional judgment about when delegation is appropriate

4. **Context Switching Prevention**
   - AI integrated into IDEs reduces productivity-killing interruptions
   - Preserves mental flow state and attention capacity

**Notable Quote:**
"The ultimate goal of a developer is to translate to a language that the computer can understand instructions to actions that will have impact in the real-world."
